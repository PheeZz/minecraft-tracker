# Онбординг: coachmark-тур + точечные подсказки — дизайн

Дата: 2026-05-31. Затрагивает оба трекера (деревья, пчёлы) + общий слой `src/shared/ui`.

## Проблема и цель

Оба экрана — плотные графовые UI. Инструмент планируется **раскатывать на реальных
пользователей**, поэтому нужен полноценный онбординг, а не просто текстовая плашка.

Делаем **два слоя** с разными ролями:

1. **Coachmark-тур** (по требованию + авто при первом заходе на вкладку) —
   пошаговая подсветка элементов с поповерами: «вот рейл, вот фильтры, вот лучший
   следующий шаг». По одному туру на домен.
2. **Точечные подсказки `HintSpot`** (только по запросу) — маленькие «?» рядом с
   ключевыми зонами; клик → короткий поповер. Пассивный справочный слой, не лезет сам.

Связка осмысленна: тур = первичная ориентация (один раз/по кнопке), хинты =
справка-по-требованию. Автоматически всплывает только тур, и только раз на домен.

## Ключевые решения (зафиксированы на брейншторме)

- **Движок тура — `driver.js`** (~5–6 KB gzip, ноль зависимостей, MIT),
  **грузится лениво** (динамический import, отдельный чанк — как ELK). На бандл
  первой загрузки не влияет.
- **Якоримся к стабильному UI-хрому**, а где нужно указать на ноду графа —
  используем DOM-карточки node-html-label (`.node[data-id]` у деревьев; у пчёл
  добавим `data-id`). Перед подсветкой ноды **программно зумимся/подсвечиваем её тем
  же API, что и клик** (`focus(id)` + `highlightLineage(id)`), ждём окончания
  пан/зума, затем driver.js спотлайтит карточку.
- **Хинты — лёгкий собственный поповер** (без затемнения экрана), т.к. для одной
  короткой подсказки спотлайт-оверлей driver.js избыточен.
- **Персист**: `onboard.trees` / `onboard.bees` (булев «тур пройден/закрыт») через
  обёртку `@/shared/persistence/storage`. Хинты состояние не хранят.
- **a11y / reduced-motion**: driver.js даёт фокус-менеджмент и клавиатуру; при
  `prefers-reduced-motion` отключаем анимацию подсветки/скролла.

## Зависимость

Добавить `driver.js` в `dependencies`. Импортировать **только лениво** внутри
`useTour` (`const { driver } = await import('driver.js')` + стили
`import('driver.js/dist/driver.css')`), чтобы код тура и его CSS попали в отдельный
чанк, а не в начальную загрузку.

## Архитектура

### `src/shared/ui/useTour.ts` (общий движок)

Composable-обёртка над driver.js. Изолирует домены от деталей библиотеки.

```ts
export interface TourStep {
  /** CSS-селектор или функция, возвращающая элемент (для динамических нод). */
  element: string | (() => Element | null)
  title: string
  text: string
  side?: 'top' | 'bottom' | 'left' | 'right'
  /** Действие ДО подсветки шага (напр. зум к ноде); ждём резолва промиса. */
  before?: () => void | Promise<void>
}

export interface TourController {
  start: () => Promise<void> // лениво грузит driver.js, запускает шаги
  destroy: () => void
}

export function useTour(steps: TourStep[], opts?: { onDone?: () => void }): TourController
```

Поведение `start()`:

1. `const { driver } = await import('driver.js')` (+ css). Кешируем модуль.
2. Учитываем `prefers-reduced-motion` → `animate: false`, `smoothScroll: false`.
3. Для каждого шага: если есть `before`, driver вызывает наш хук
   (`onHighlightStarted` → `await before()` → `driverObj.refresh()` чтобы
   пересчитать позицию после зума/скролла). `element` поддерживает функцию —
   для нод, чей id вычисляется на лету.
4. `onDestroyed` / завершение → `opts.onDone()` (выставит флаг персиста).
5. Кнопки/локализация поповера на русском: «Назад / Далее / Готово / Пропустить».

`destroy()` — снять driver и слушатели (вызов в `onBeforeUnmount` вью).

### `src/shared/ui/HintSpot.vue` (точечная подсказка)

Маленький круглый «?»-триггер; по клику показывает лёгкий поповер рядом с собой.
Без затемнения, без последовательности.

- Props: `text: string`, `title?: string`, `side?: 'top'|'bottom'|'left'|'right'`.
- Поповер позиционируется относительно «?» (базовый флип если не влезает по краю).
- Закрытие: повторный клик, клик вне, Esc. `aria-expanded`, `role="note"` на поповере.
- Это **собственный** компонент (driver.js не используем — он для тура).

### Кнопка запуска тура

Маленькая `?`-кнопка «Тур» в панели каждой вкладки (рядом с уже существующими
контролами). Запускает `tour.start()` в любой момент. Авто-старт — см. триггеры.

### Графовые хуки (для шага «лучший следующий выбор»)

**Деревья** (`TreeGraph.vue`) — расширить `defineExpose`:

```ts
defineExpose({
  focus: (id) => graph.focus(id),
  flash: (ids) => graph.flash(ids),
  // НОВОЕ для тура:
  tourBestId: () => availSorted().sort(byScore)[0]?.id ?? null, // вынести расчёт из bestStep
  tourSpotlight: async (id) => {            // зум + подсветка как при клике, ждём пан
    ui.selectedId = id
    graph.focus(id)
    graph.highlightLineage(id, 'ancestors')
    await graph.whenIdle()                  // см. ниже
  },
})
```

Нужен способ **дождаться окончания пан/зума**. В `useTreeGraph` добавить
`whenIdle(): Promise<void>` — резолвится после завершения анимации `focus`
(cytoscape `cy.animate({…, complete})` уже есть в focus — пробросить колбэк, либо
`cy.promiseOn('layoutstop')`/таймаут-фолбэк ~400мс). Тур: `before` шага =
`treeGraphRef.tourSpotlight(treeGraphRef.tourBestId())`, `element` =
`() => document.querySelector('.node[data-id="…"]')` (id из tourBestId, сохранённый
в замыкании шага). Если `tourBestId()` вернул null (нет доступных) — шаг
**пропускается** (стройка шагов фильтрует null-таргеты).

**Пчёлы** (`useBeeChainGraph` шаблон ноды) — добавить `data-id` в карточку
(`beeNodeTemplate`): `<div class="${cls}" data-id="${d.id}" …>`. Граф контекстный,
поэтому шаг с нодой возможен только после демо-выборки соты (см. контент тура пчёл).
Расширить экспортируемое из `BeeChainGraph`/`BeesView` минимально: тур пчёл
оперирует стором (`store.selectComb(SAMPLE_COMB)`), а ноду-цель якорит по
`.gnode[data-id="…"]` (целевой вид = `store.curTarget`).

## Контент тура

> Селекторы ниже — ориентир по смыслу зоны; точные классы/`data-tour`-атрибуты
> фиксируются на этапе реализации по реальной разметке (для надёжности добавим
> явные `data-tour="..."` якоря на целевые элементы, а не полагаемся на классы-стили).

### Деревья (`tours/treesTour.ts`)

1. **Рейл/сайдбар** (`.sidebar` или его контейнер) — «Слева — список деревьев,
   фильтры по тирам, раскладки и поиск».
2. **Фильтр тиров** (`.tier-filter` / селектор фильтров) — «Скрывай/показывай тиры».
3. **Раскладка** (`.select` раскладки) — «Меняй раскладку графа (tiers/ELK/dagre)».
4. **Навбар графа** (`.stage__navbar`) — «Кнопки: следующий доступный, лучший шаг,
   вписать, все рёбра».
5. **Лучший следующий выбор** (динамическая нода) — `before`: зум+подсветка лучшей
   доступной ноды; «Вот лучший следующий шаг — больше всего открывает дальше».
   *(пропускается, если доступных нет)*
6. **Карточка ноды** (та же или соседняя `.node`) — «Клик по ноде — родословная;
   значок-инвентарь — счётчики саженцев/пыльцы».
7. **Легенда** (`.legend`) — «Что значат цвета/обводки».

### Пчёлы (`tours/beesTour.ts`)

1. **Модбар** (`.modebar`) — «Три режима: Граф, Инвентарь, Задачи».
2. **Рейл** (`.rail`) — «Слева выбираешь соту или пчелу; поиск, переключатель
   Соты/Пчёлы».
3. `before` следующего шага: `store.selectComb(SAMPLE_COMB)` — демонстрационно
   выбрать соту, чтобы появились граф и план.
4. **Граф цепочки** (`.cy` / целевая нода `.gnode[data-id]`) — «По центру — дерево
   скрещивания выбранной цели».
5. **Панель плана** (`.panel`) — «Справа — пошаговый план: кого с кем скрестить».
6. **Кнопка Задачи** (`.modebar__tasks`) — «Задачи по сотам: что нужно для крафта».

`SAMPLE_COMB` — стабильная сота с коротким планом (напр. «Медовые соты» — есть
производитель глубины 0). Хранить как константу тура.

## Триггеры и персист

- **Авто-старт**: при первом заходе на вкладку, если `storage.get('onboard.<id>',
  false) === false`. Старт после готовности графа (деревья — после `onReady`;
  пчёлы — сразу, граф для шагов появляется по демо-выборке). Небольшая задержка,
  чтобы UI отрисовался.
- **Завершение/пропуск** тура → `storage.set('onboard.<id>', true)` (через `onDone`).
- **Кнопка «?» (Тур)** в панели — запускает тур всегда, игнорируя флаг.
- Хинты доступны всегда, флага не имеют.

## Состояния и крайние случаи

- **Нет доступной ноды для шага «лучший выбор»** (всё выведено/ничего не доступно) →
  шаг исключается при сборке списка (фильтр по `tourBestId() != null`).
- **Пчёлы: граф пуст до выбора** → демо-`selectComb` перед графовыми шагами; по
  завершении тура НЕ откатываем выбор (пользователь видит реальный пример).
- **Resize/scroll во время шага** → `driverObj.refresh()` (driver.js сам слушает, но
  после нашего зума вызываем явно в `before`/после анимации).
- **KeepAlive**: вью кешируются; авто-старт вешаем на `onMounted`/`onActivated` с
  проверкой флага, чтобы при возврате на вкладку тур не перезапускался.
- **Тур прерван закрытием вкладки/Esc** → считаем как «пройден» (ставим флаг), чтобы
  не доставать пользователя; перезапуск — по кнопке.
- **reduced-motion** → без анимаций подсветки/скролла; зум к ноде — мгновенный.

## Тестирование

- **Юнит**: сборка списка шагов (фильтрация null-таргетов; reduced-motion флаги).
- **Playwright (chromium + webkit)**:
  - первый заход на деревья → тур авто-стартует; «Далее» проходит шаги; шаг «лучший
    выбор» зумит и подсвечивает `.node[data-id]`; «Готово»/«Пропустить» ставит флаг;
    повторный заход → тура нет; кнопка «?» запускает снова.
  - пчёлы: тур стартует, демо-`selectComb` наполняет граф/план, шаги по
    рейл/граф/панель/задачи.
  - `HintSpot`: клик открывает поповер, Esc/клик-вне закрывают.
  - driver.js-чанк **не** грузится до старта тура (ленивость), грузится при старте.
  - нет ошибок консоли.

## Вне рамок (YAGNI)

- Многоязычность тура (пока только русский).
- Ветвящиеся/условные туры, прогресс «N из M» между сессиями.
- Подсветка отдельных нод в пчелином графе сверх целевой.
- Аналитика прохождения онбординга.
